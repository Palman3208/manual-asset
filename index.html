behavior

This variable is responsible for the behavior of the entity. There are only 4 modes in the standard package:

  • player
  • AI_lame
  • AI_medium
  • AI_pro
  • AI_example

The behavior of the player turns the bot into a key-controlled WASD entity. The controls are extremely simple:
        WASD -movement
        shift - sneak
        LMB - kick
        control - put block. 
        To successfully put a block - point the mouse at the opponent so that the line from you to the mouse
coincides with line up from you
 to the attacker and press control.
        The block lasts only 30 frames and recharges the same amount. With a successful block, the delay disappears
and you can put it back on immediately.
       Under standard conditions
you will be thrown some distance away, but if you put a block at the last moment, you will parry the enemy,
provided the can_parry variable is activated. The enemy
it will throw him back, and during the rollback he will receive triple damage from melee attacks. 
       Other items in
 the list - AI levels. AI can be changed in the ai_behavior script, and the items in the variable editor. 
       IMPORTANT NOTE!!! The items are written in a strings, so you should write the names inin quotation marks, for example "AI_lvl_1".

team

The team variable is also presented as a list of possible options. It represents a label for entities. There are three points:

  • team_player
  • team_enemy
  • peaceful_penguin

In the standard package, if the team variable is similar to yours, the entity will become your ally, and if it is different, it will become your enemy accordingly. But not with the peaceful_penguin clause. These guys will never attack you, or the enemies, or them they will never pay attention, and in the case of they won't get angry at an accidental hit :3. You can set yourself any item, then entities with they won't attack you. It can also be changed during development and the contents of the list can be changed in the object editor.

    • slash_attack()

    This function forces the selected object to attack in a certain direction, and it does not matter whether the entity is it or a completely different object
    parent_id (default caller id) attacker id
    trg_x/y (default mouse position) this is the position that the attack will be directed at
    damage (default is 30) the damage inflicted by the attack

    sprite (by default _spr_slash) the sprite used by the target of the attack
    blockable (default is true) is it possible to block this attacker
    hit_power (default 8) attack drop power
    hit_time (default is 8) the duration of the attack drop
    sound_hit (default is noone) the sound played when the entity is hit. Can support both arrays, so are single sounds
    sound_miss (default is noone) the sound played when the attack misses. It can support both arrays and single sounds.

    Usage example

    if( mouse_check_button_pressed(mb_left)){
          slash_attack()
    }

    How to trigger an attack for a bot?

    There are two ways
    The first is using the slash_attack function, filling in all the fields
    This function makes any object attack instantly, suitable for the player
    The second one is using the push_attack_type_slash() function
    This function adds an attack to the array. You can try to make combinations.
    Works only for bots

    colorr - the color of the attack indicator, if you don't need it - write noone
    timer - the time in milliseconds when the enemy will launch an attack
    trg is the x, y coordinate in the array, for example [0,0]. If you insert the id of an existing object instead of an array, the attack will be directed at him

    attack_type - variable responsible for types of attacks

    1 - Melee attack
    2 - throw a projectile
    3 - make the attack an "explosion"
    4 - dash

    Types of attacks will also be introduced in the future, for example laser attack

    hit_animation - sprite animation played by the animate function.
    If it is set to false, there will be no animation
    Otherwise, three more variables should be added to the command

    hit_animation_down - the sprite used by the object after the attack is completed in the downward direction
    hit_animation_up - the sprite used by the object after the attack is completed in the upward direction
    hit_animation_side - the sprite used by the object after the end of the attack in the direction of the side

    bullet_attack()

    The function with the largest number of variables.
    Releases a projectile in the direction of the target.


    parent_id (default caller id) attacker id
    trg_x/y (default mouse position) target for bullet
    damage (default is 5) the damage inflicted by the bullet
    sprite (by default spr_bullet) a sprite for a projectile
    life_time (default is 10) bullet flight time
    bullet_speed (default is 30) bullet flight speed
    sound_start (no by default) the sound played at the start of the flight
    sound_end (no by default) the sound played at the end of the flight
    sound_slam (no by default) the sound played when a bullet collides with a wall
    sound_fly (no by default) looped sound played at the start of the flight
    sound_hit (no by default) the sound played when a bullet collides with an entity
    spread (default 0) bullet spread
    The standard command to call an attack for a bot

    push_attack_type_bullet()

    Usage exmple

    if( mouse_check_button_pressed(mb_right)){
          billet_attack()
    }

    blast_attack()

    Deals damage to ALL nearby entities, causing them to fly away at a given speed
    Has several variables:

    parent_id (default caller id) attacker id
    radius (default 300) the distance in pixels at which entities will take damage
    damage (default is 30) the damage inflicted by the attack
    sprite (no by default) the sprite used by the target of the attack
    hit_time (default is 20) the departure time of the attacked entity
    blast_power (default 30) the force with which the entity will fly away from the attacker
    energy_sound (default is noone) sound when an explosion is activated
    blast_sound (default is noone) sound when exploding
    The standard command to call an attack for a bot

    push_attack_type_blast()

    Usage example

    if( keyboard_check_pressed(vk_alt)){
          blast_attack()
    }

    push_attack_type_dash()

    WORKS ONLY FOR BOTS ! Launches the entity to the specified coordinates/targets
    Has several variables:

    dashtype (default 2) is the type of dash. 1 - jump to the coordinates without stopping in a straight line until the end of the timer
    2 - a dash to the coordinates with a stop at the target. If an id is indicated, it moves to it
    3 - a dash to the coordinates with a stop at the target in a straight line until the end of the timer

    dashspeed (default is 30) dash speed
    dashtarget (by default [target.x, target.y]) the departure time of the attacked entity
    dashtime (default is 30) duration of the dash in frames

    nearest_enemy(max_distance)

    Search for the nearest enemy at a given distance. Does not react to entities outside the walls

    animate(sprite_down, sprite_up, sprite_side, animation_speed)

    Changes the sprite to the specified one depending on the direction variable

    walk_towards_object(target_id,speed)/run_towards_object(target_id,speed)

    Uses mp_grid (64/64) technology and directs the entity to the target.
    target_id - id of the target to which the entity will be go
    speed (by default, the standard speed of the entity) - the speed at which the entity will go to the goal

    stop()

    If you need to stop an entity, this function is perfect for you. Stops animations and movement along the paths.
